Documentation Hub

Welcome to the official documentation for the Nepalgunj Skin Center website. This guide explains the architecture, content specificities, and deployment processes for the developer team.

Note: This site uses a Static Site Generation (SSG) approach. Changes should always be made in the src/ directory.

Project Overview

The Nepalgunj Skin Center website serves as the digital front door for Western Nepal's premier aesthetic clinic. It allows patients to explore services, read testimonials, and contact the clinic.

Key Goals

  • Inform: Provide detailed info on Hair Transplants, Laser treatments, etc.
  • Trust: Showcase Dr. Abhishek Arjel's credentials and patient success stories.
  • Convert: Drive bookings through prominent "Book Now" calls to action.
  • Perform: Ensure fast load times on all devices using static HTML.

Technology Stack

Category Technology
Core HTML5, CSS3, JavaScript (ES6+)
Build Tool Node.js (Custom Build Script)
Styling LESS (Pre-processor), Bootstrap Grid
Interactive jQuery, Revolution Slider, Owl Carousel

Directory & File Structure

We separate source code from built code. Always edit files in src/.

Warning: Do not edit the .html files in the root directory directly. They are overwritten by the build script. 
├── src/                # SOURCE CODE (Edit here!)
│   ├── header.html     # Global Header
│   ├── footer.html     # Global Footer
│   └── *.html          # Page templates
├── css/                # Stylesheets
│   └── style.css       # Main compiled CSS
├── js/                 # JavaScript
│   └── custom.js       # Main logic file
├── images/             # Assets
│   ├── hosp/           # Hospital photos/logo
│   └── patient/        # Patient photos
├── build.js            # Build Script
└── index.html          # GENERATED FILE (Output)

Key Directories

  • src/: The "Workspace". All editing happens here.
  • images/: Organized by type (e.g., hosp for infrastructure, patient for clinical results).
  • css/: Contains style.css (the main look and feel) and bootstrap.min.css.

Pages Guide

A breakdown of the website's key pages and their components.

1. Homepage (index.html)

The landing page features:

  • Hero Slider: Large rotating banner (Revolution Slider).
  • Concerns Grid: Icons linking to specific treatments (Acne, Aging, etc.).
  • Success Stories: A marquee slider containing patient testimonials.

2. About Us (about.html)

Contains the clinic's Mission, Vision, and detailed profiles of the medical team (Dr. Abhishek Arjel).

3. Services (services.html)

The catalog of treatments. Each service item links to booking or further details.

4. Gallery (gallery.html)

Before & After photos. Uses a lightbox plugin to view images in full screen.

5. Contact (contact.html)

Includes:

  • Google Map embed (Pasang Lhamu Road).
  • Contact Form (Name, Email, Phone, Message).
  • Direct phone links for mobile users.

Setup & Deployment

Prerequisites

You need Node.js installed to run the build script.

Local Development

  1. Clone the repository.
  2. Open terminal in the project root.
  3. Run the build command whenever you change a file in src/:
node build.js

This compiles the templates in src/ into the final HTML files in the root folder.

Deployment

The site is static, so it can be hosted anywhere (Vercel, GitHub Pages, Netlify, Apache/Nginx).

Recommended (Vercel): Simply push to GitHub and import into Vercel. Vercel will automatically detect the static files.

SEO Strategy

The website is pre-optimized for Search Engines to identify it as the leading skin center in Nepalgunj.

Meta Tags

Every page includes unique meta tags in the <head> section.

  • Title Tag: Includes primary keywords (e.g., "Advanced Dermatology", "Hair Transplant").
  • Description: A compelling summary displayed in search results.
  • Keywords: Targeted phrases like "dermatologist nepalgunj", "laser clinic".

Open Graph (Social Media)

When shared on Facebook, LinkedIn, or Twitter, the site displays a rich preview card.

Note: The default sharing image is located at images/hosp/logo.png.

Structured Data (Schema.org)

We use JSON-LD structured data to tell Google explicitly about the business details.

{
  "@context": "https://schema.org",
  "@type": "MedicalBusiness",
  "name": "Nepalgunj Skin Center",
  "address": {
    "streetAddress": "Pasang Lhamu Road",
    "addressLocality": "Nepalgunj",
    ...
  }
}

This increases the chance of showing up in the Google "Local Pack" (Map results).

Technical Reference

Namespace

All custom JavaScript logic is encapsulated in the THEMEMASCOT object to prevent global namespace pollution.

See js/custom.js for the implementation.

Key Libraries

  • Revolution Slider: Used for the complex animations in the hero section.
  • Magnific Popup: Handles all lightboxes (images, galleries, videos).
  • Owl Carousel: Used for the testimonial sliders.

CSS Customization

The site uses LESS for pre-processing. While you can edit css/style.css directly for quick fixes, the source of truth is in the LESS files (if available in the repo).

Theme Colors: Look for --primary-color variables or hex codes in style.css to adjust branding.
